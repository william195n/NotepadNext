3. 自动补全触发阈值硬编码
src/decorators/AutoCompletion.cpp:51

if ((curPos - startPos) < 3)  // 硬编码的 3 字符
    return;
问题：最小触发字符数硬编码为 3
建议：移到设置中，让用户可配置（1-5 字符）

4. 缺少大文件保护机制
src/ScintillaNext.cpp:626

// TODO: figure out what to do if "size" is too big
allocate(file.size());
问题：没有检查文件大小上限，可能导致内存耗尽
建议：添加文件大小警告（如 > 100MB 提示用户）

5. UTF-16 编码未完全支持
src/ScintillaNext.cpp:666-668

if (bomType == BomType::Utf16BE || bomType == BomType::Utf16LE) {
    // Um...ignore this for now?
}
问题：UTF-16 文件检测到但未处理
建议：要么实现转换，要么明确提示用户不支持

6. 内存管理
现状：代码中大量使用 new 分配（112 处），大部分依赖 Qt 父对象管理
风险：部分对象如果父对象设置不当可能泄漏
建议：考虑使用 std::unique_ptr 或 QScopedPointer

7. 错误处理不一致
问题：有些地方用 qWarning() 记录错误但继续执行，有些地方返回 false
建议：统一错误处理策略，特别是文件操作失败时
💡 用户体验改进建议
