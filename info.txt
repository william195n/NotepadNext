是的，我确认 NotepadNext 的 GitHub 仓库中配置了自动编译脚本 (GitHub Actions)。

在项目的 .github/workflows/build.yml 文件中，定义了在每次代码推送（Push）或提交拉取请求（Pull Request）时，都会自动在 Windows、Linux 和 macOS 上进行编译，并上传以下产物：

NotepadNext-Windows-Qt6.5-Zip: Windows 免安装压缩包。
NotepadNext-Windows-Qt6.5-Installer: Windows 安装程序 (.exe)。
Linux AppImage: Linux 可执行镜像。
macOS dmg: macOS 安装镜像。
如何找到这些编译好的文件？
访问项目的主页：https://github.com/dail8859/NotepadNext
点击上方的 Actions 选项卡。
在左侧选择 Build Notepad Next 任务。
点击最近的一个成功的任务记录（通常带绿色对勾）。
滚动到页面底部的 Artifacts 区域，你就可以直接下载编译好的压缩包或安装程序。
⚠️ 重要提示
这些自动生成的产物是针对官方源码的。 由于我们刚才在本地修改了代码，官方的 GitHub Actions 不会自动包含我们的修改。

如果你想获得一个**包含我们刚才修改的功能（不区分大小写高亮）**的可执行文件，你有两个选择：

本地编译：按照我之前提供的步骤（安装 VS2022 和 Qt）在你的电脑上编译。
利用 GitHub 自动编译：
在 GitHub 上 Fork 这个仓库到你自己的账号下。
把你本地修改后的代码 Push 到你的 Fork 仓库中。
你自己的仓库里的 GitHub Actions 会自动开始编译，完成后你就能在你自己的 "Actions" 页面下载到包含新功能的安装包了。
